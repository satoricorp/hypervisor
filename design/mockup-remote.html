M8b — iMessage bridge. hypervisor reads the self-chat from chat.db (read-only, like the cursor adapter) and replies via Messages. approvals are off by default over this channel — identity is softer than the tailnet. tap the suggestions to see the conversation.

Threat model

a remote channel controls approvals = command execution on the mac. so: tailnet = full control (WireGuard + device identity), iMessage = read-mostly (status, prompts; approvals opt-in), and no channel ever exposes yolo.

Plumbing

the mobile page and SSE stream reuse the exact snapshot + command layer the desktop uses — no second code path. every remote action toasts on the desktop and lands in history: approved via remote · joe@… · 14:02.

Sequencing

after M7 (menu bar + notifications), which builds the command-from-outside plumbing. order: M8a tailnet → M8b iMessage. spec: design/remote.md.
