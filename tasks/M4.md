# M4 — worktrees

**Prerequisite gate:** none hard (independent feature). Builds on the spawn path
(M2) and the settings pattern (REALIZE/POSTHOG). Written + built 2026-07-12.

## Goal (PLAN.md)

`/worktree`, auto-worktree default when `/new` targets a repo with an active
session, repo·branch·worktree line in the header, settings toggle.

**AC:** two sessions in one repo never share a working tree unless the user
opts out.

## What was built

Backend:
- `control/worktree.rs` — `repo_root(cwd)` resolves the *shared* repo identity
  via `git rev-parse --git-common-dir` (same for the main tree and any linked
  worktree), `add(root)` runs `git worktree add <sibling> -b hv-<id>`, sibling
  dir `<repo>.hv-<id>`. Unit-tested against a real temp git repo.
- `spawn_session(… worktree: Option<bool>)` — `Some(true)` (`/worktree`) forces
  isolation; `None` (`/new`) auto-isolates when `settings.auto_worktree` and the
  repo's **main tree** already has a live session (`main_tree_busy` →
  `is_in_main_tree`). Git failure: explicit → error, auto → toast + share tree.
- `owned::Worktree { repo, branch, path }` persisted on `OwnedEntry` +
  `PendingOwned`; `SessionWire.worktree` populated in `to_wire`/`merge_pending`,
  overriding repo/branch to the shared repo + dedicated branch.
- `settings.auto_worktree` (default **on** — isolate unless opted out).
- `telemetry` `CommandName::Worktree`.

Frontend:
- `menuData.ts` `/worktree`; `store.tsx` `doWorktree` (inherits the selected
  session's harness+model, forces a worktree) + a filter that shows it only for
  a spawnable repo session; `api.ts` `spawnSession(… worktree)`.
- `wire.ts` maps `worktree → s.wt`; header renders `⎇ repo · branch · worktree`
  (was a dead placeholder). Settings toggle row; removed the "returns with M4"
  footnote.

## Scope fence

- Worktrees are created only on an explicit spawn — never on scan/tick (H2 hot
  loop untouched; no per-tick git). Adapters stay read-only.
- Sibling `<repo>.hv-<id>` dirs only; no writes inside harness config dirs.
- No worktree pruning/GC in v1 (idle-kill of `hv-*` tmux already exists; leftover
  worktrees are the user's to `git worktree remove`). Note for a follow-up.
