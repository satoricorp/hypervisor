# M6 — usage + cost ledger

**Gate:** none hard. Access view (env/keychain presence) already shipped in
REALIZE; this milestone adds the **token + cost ledger**. Built 2026-07-13.

## Goal (PLAN.md)

Token/cost ledger from transcripts (claude + codex record usage; cursor
best-effort), pricing table shipped + updatable, subscription-vs-API split,
Access view. **AC:** ticker `$x.xx · x.xM TOK` bottom-left matches a hand-check
of one day.

## What was built

- `src-tauri/src/usage.rs` — on-demand ledger (never the 2s tick; reads
  transcript files):
  - `price_for(model)` table in USD/Mtok (opus/fable, sonnet, haiku, gpt-5/codex,
    o4-mini; unknown → 0, tokens still counted). Single source of truth,
    documented as hand-editable.
  - claude: sums `message.usage` (`input`, `cache_creation`, `cache_read`,
    `output`) per assistant line, priced per that line's `model`, day-bucketed
    by the line `timestamp` (local calendar).
  - codex: last `total_token_usage` per rollout, best-effort model, day by mtime.
  - `scan()` → `UsageReport { today_cost, today_tokens, total_cost,
    total_tokens, rows[(harness,model)], api_priced }` within a 30-day / 200-file
    window.
- `get_usage` tauri command.
- Ticker (`Statusbar.tsx`) now shows today's **`$x.xx · x.xM tok`** (polls every
  120s; falls back to live counts before first load).
- Usage view (`UsagePane`) — Today / Last-30-days tiles + per-model rows with
  cost in mint. Replaces the counts-only placeholder.

## Subscription-vs-API

Cost is the **API-equivalent** price of the tokens. The Access view (REALIZE)
already shows whether a harness is on a subscription vs a live API key;
subscription-covered sessions therefore read as their api-equivalent cost, noted
in the pane footnote. A true $0-for-subscription split needs per-session key
attribution — deferred.

## Scope fence

- On-demand only; the H2 hot loop and read-only adapters are untouched.
- No network price fetch — the table ships in-binary and is edited in source.
- cursor + opencode omit token usage (best-effort per PLAN).
