**You are building exactly one thing this session:** `cargo test --lib`
becomes safe to run on any machine (no live side effects, no panics when no
session data exists), and five small correctness bugs from the 2026-07-10
architecture review are fixed.

Not in this session: the 2s-poller rework (H2), UI/toast changes (H3),
grammar (M7g). Adapter parsing untouched — `compare.py` is the tripwire.

## Definition of done

1. `cargo test --lib` passes WITHOUT adopting any real session, sending any
   prompt to any real session, or launching any harness CLI. Proof:
   `tmux -L hypervisor ls` before/after the run shows no new sessions, and
   no new claude/codex/opencode transcript files appear during it.
2. The live-fire tests still exist, but are `#[ignore]`d and additionally
   require `HV_LIVE=1`; run via `HV_LIVE=1 cargo test --lib -- --ignored`.
   Without the env var they skip with an explanatory eprintln.
3. An opencode api-tier session accepts two prompts within 60s from the
   prompt bar — the idle guard is gone from the api path. The adopt fork
   guard (claude/codex `--resume` genuinely forks) is untouched.
4. Codex sid correlation works across midnight: `find_codex_sid` scans
   today's AND yesterday's date dirs (unit test with a fixture dir; no live
   requirement).
5. An owned session's permission dialog is detected even when the session
   is absent from the top-8 snapshot or sits in `done` state: owned.json v2
   records harness alongside the tmux name; `detect_tmux` reads harness
   from the owned map (snapshot only as fallback) and no longer skips
   `done`; entries whose tmux session no longer exists are pruned on load.
6. `python3 spike/compare.py` OK · `bunx tsc --noEmit` · `cargo test --lib`
   · `npm run tauri dev` boots.

## Steps

1. **Test gating.** Mark `#[ignore]` + require `HV_LIVE=1` on:
   - `control::adopt::tests::fork_guard_and_adopt_idle_sessions` — adopts
     REAL user sessions via `claude --resume` and panics when no idle
     session exists. This is the worst offender.
   - `control::opencode::tests::live_opencode_idle_guard_prompt_and_new` —
     sends a real prompt to the user's most recent opencode session.
   - `control::tmux::tests::opencode_spawn_arm_builds_tmux_session` —
     launches the real opencode binary.
   Pure-logic tests (pane parsers, uuid recovery, message shapes) stay
   ungated. `cursor_watch_only_and_tmux_socket_works` needs only tmux +
   `sleep` — keep ungated; tmux is a documented prereq.
2. **Opencode idle guard.** In `events.rs::send_prompt`, delete the 60s
   guard on the opencode branch. Rationale: HTTP is opencode's intended
   concurrent-access path — nothing forks; the guard throttled normal
   conversation to one message per minute (mtime refreshes on every agent
   reply).
3. **Codex midnight.** `owned.rs::find_codex_sid` checks today's and
   yesterday's `~/.codex/sessions/<Y/m/d>` dirs.
4. **owned.json v2.** Value becomes `{"tmux": "hv-…", "harness": "…"}`.
   `load()` accepts the old plain-string form (harness unknown → fall back
   to snapshot lookup as today); saves write v2. Prune entries whose tmux
   session is gone (`tmux -L hypervisor has-session`) at load time —
   a dead pane can't receive approvals anyway.
5. **Yolo persistence DECISION.** Yolo stays in-memory deliberately —
   a forgotten yolo must not auto-approve anything after a restart. Write
   the `// DECISION:` comment at the flag declaration in `AppState`.
6. **Repo hygiene.** Delete `bun.lock` (npm is the scripted path). Write a
   real README: prereqs (macOS, tmux, Rust toolchain, node/npm, harness
   CLIs resolvable from a zsh login shell, `claude /login` refreshed),
   run (`npm run tauri dev`, port 1420 is strict), headless checks
   (`cargo run --bin hvscan`, `hvscan --watch`), tests (`cargo test --lib`
   vs `HV_LIVE=1 cargo test --lib -- --ignored`).
