## The contract: design/mockup-b.html

Open it in a browser and drive it before writing code. Port it ‚Äî do not
redesign it. Specifically:

- **CSS**: copy the `:root` tokens and component styles into `src/styles.css`
  nearly verbatim. Same colors, radii (window 10 / cards 3 / chips 2), type
  sizes, `font-variant-numeric: tabular-nums`. Font stack stays
  `"Berkeley Mono", "TX-02", ui-monospace, "SF Mono", Menlo, monospace`
  (local fonts; no webfont downloads). Wordmark font-family starts with `"Xer0"`.
- **DOM structure**: mirror the mockup's structure and class names
  (`.titlebar`, `#side`, `.rightcol`, `#main`, `.promptzone`, `.statusbar`,
  `.srow`, `.dlog`, `.cmdmenu`, `#palette`, `#toast`). This keeps the CSS port
  mechanical and diffable.
- **Behavior**: the mockup's `<script>` is the behavioral spec ‚Äî selection,
  subSel, view switching, the command-menu state machine (`root ‚Üí target ‚Üí
  model`), approvals, yolo, adopt, the 3.2s thinking-line cycler. Port that
  logic into React state; don't invent new flows.

## Steps

1. **Window chrome.** In `tauri.conf.json`: width 1180, height 740, min 900√ó560,
   `"titleBarStyle": "Overlay"`, `"hiddenTitle": true` (macOS native traffic
   lights over our dark titlebar). Delete the mockup's fake `.desk`/`.stage`
   backdrop and fake `.lights` ‚Äî the app window replaces them; keep the
   titlebar row but give it `padding-left: 78px` so native lights don't overlap
   the wordmark.
2. **State.** One store via `useReducer` + context (no state libraries). Shape:
   `{ sessions: Session[], sel, subSel, view, menu, palette, yolo, toasts }`.
   Copy the mock `sessions` array from the mockup script into
   `src/mockSessions.ts` unchanged (types in `src/types.ts` per PLAN.md ¬ßData
   model ‚Äî note the UI type uses camelCase; M2 maps the Rust snake_case).
3. **Components.** `src/components/`: `Titlebar`, `Sidebar` (+ `NewAgentButton`),
   `MainPane` (switches session / subagent / usage / access / settings /
   history), `Transcript`, `SubagentList`, `PromptBar`, `CommandMenu`,
   `Palette`, `Statusbar`, `Toast`. Views' content = port of the mockup's
   `PANES` HTML.
4. **Keyboard.** One global `keydown` listener (in a hook), porting the
   mockup's handler order exactly: ‚åòK first ‚Üí palette-open branch ‚Üí
   other-input guard ‚Üí esc chain ‚Üí menu-open branch ‚Üí prompt-focused branch ‚Üí
   Tab approve ‚Üí digits ‚Üí j/k ‚Üí h/l ‚Üí ‚èé ‚Üí `/` ‚Üí any-letter-focuses-prompt.
   ‚åòN triggers the New Agent flow (works in Tauri; browsers reserve it).
5. **Timers.** Port the fake async: send ‚Üí working ‚Üí done (4‚Äì7s), subagent
   spawn ‚Üí done (6‚Äì9s), thinking-line cycler (3.2s). Keep them in one module
   (`src/simulate.ts`) so M2 can delete them cleanly.
