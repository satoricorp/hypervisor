# M5 — history + memory

**Gate:** none hard. Built 2026-07-13. Replaces the interim history (REALIZE)
with a persistent sqlite summary store.

## Goal (PLAN.md)

sqlite store; archive-time summary (outcome + files touched); history view =
search over summaries; `/new` attaches top related summaries (same repo first,
then embedding similarity). **AC:** create a new agent in a repo with prior
sessions; its first context message contains those summaries, and the history
view finds them by keyword.

## What was built

- `src-tauri/src/history.rs` — sqlite at `app_data_dir/history.db` (opened
  per-op, on-demand): `summaries(sid PK, harness, repo, cwd, title, summary,
  files, archived_at)` + repo index. `upsert` / `get` / `search` (keyword LIKE
  over title/summary/files/repo) / `same_repo` (newest first, excludes self).
- **Extractive summarizer** (`summarize`) — last non-empty assistant message
  (outcome) + distinct Edit/Write file paths, into one line. No model call →
  content never leaves the machine, always available. `context_message` formats
  same-repo summaries for a new agent.
- **Archive hook** — `archive_one` (both `/archive` and `/archive idle`) now
  calls `store_summary`: reads the session's transcript, summarizes, upserts.
  Best-effort — a summary failure never fails the archive.
- **`/new` context attachment** — `spawn_session` looks up same-repo summaries
  and, after the pane is ready (~3.5s), sends them as the new agent's first
  message (`context_message`). Skipped when history is empty.
- **History view** — `HistoryRow.summary` populated from the store; the search
  box now hits summaries and, when a query is present, queries the whole store
  via `search_history` (not just the recent window), unioned with the instant
  client filter. Rows show the summary.

## Deferred (documented follow-up)

Neural embedding similarity (HelixDB embedded / turbopuffer opt-in) is **not**
in v1 — retrieval is **same-repo + keyword**, which meets the AC. Embeddings +
a vector backend (off by default per principle 3) are a clean follow-up
milestone. Also: cheap-model summaries via the user's key (extractive-only for
now); the `/new` context is sent as a first message (a first-prompt prepend
would avoid a rare interleave race).

## Scope fence

- Local-only sqlite; never touches harness dirs. On-demand (open per op) — the
  2s tick and read-only adapters are untouched.
- Summaries are content-derived but stay on disk; nothing is sent anywhere.
