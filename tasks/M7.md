## Build order within this milestone

### 1. The grammar parser — backend, once, first

`src-tauri/src/grammar.rs`: `parse(text) -> Command` where Command is
`Status | Approve(letter) | Prompt(session_n, text) | Nudge(session_n) |
Unknown(hint)`. Letters map to the pending-approval queue: **letters are
assigned in arrival order, stay stable while visible, recycle only after
resolution** (a queue that re-letters mid-glance causes mis-approvals — this
is a locked design rule). Expose `run_command(text) -> String` (the reply
text) as the single entry point; the dropdown, ⌥Space, and later M8a/M8b all
call it. Unit-test the parser (`a`, `B`, `3: fix the test`, `status`,
`nudge 6`, garbage → hint).

### 2. Menu bar item + dropdown

- Tray icon via tauri `TrayIconBuilder`. Icon asset: the Xer0 "H" mark as a
  template PNG. Generating it: render the glyph with the locally-installed
  Xer0 font (small swift/CoreText or Python/CoreText script — record the
  script in Evidence). If font rendering proves unscriptable, ask Joe to
  export the glyph from a font tool once and commit it as
  `src-tauri/icons/tray-template.png`; a plain "H" placeholder is the last
  resort (note it).
- Aggregate state: overlay the corner status dot by compositing at runtime
  (three pre-rendered variants — red/yellow/green — switched on snapshot
  changes is simplest and fine). Tray title text shows the red count.
- Click → a chromeless dropdown **window** (not an NSMenu — we need inputs)
  anchored at the tray icon position (tray click events carry the rect).
  Contents per the mockup: dots summary + yolo state, approval rows with an
  Approve button, compact session rows (click → focus main window on that
  session), grammar input at the bottom. Reuse the main window's CSS tokens.
- `ActivationPolicy::Accessory` when the main window closes (menu bar app,
  no zombie dock icon) — back to `Regular` when reopened.

### 3. Notifications

- On a new pending approval: notification `Hypervisor — A · 5 wants:
  Bash(…)`. Buttons: **Approve** + **Reply…** (inline text = deny with
  guidance). This needs `UNUserNotificationCenter` category actions
  (`UNNotificationAction` + `UNTextInputNotificationAction`) — the basic
  tauri notification plugin cannot do it; use objc2/objc bindings (the
  pattern for native calls is in tv.rs). **Fallback if actions prove
  unreachable in reasonable time:** the notification body-click opens the
  ⌥Space bar pre-filled with the pending letter — degrade, don't drop; record
  which path shipped.
- Done/stalled notifications: opt-in per event type in Settings, batched
  ≤1 per 30s per session.

### 4. ⌥Space bar

- tauri global-shortcut plugin, `alt+space`. A small chromeless centered
  window per the mockup: board strip, one input, transient output line, esc
  closes. Input → `run_command` → show the reply. This is the notification
  fallback target and the desktop's single-letter approve path.

### 5. The rest

- Dock badge = red count (tauri v2 badge API on macOS if present in our
  version; else objc `NSApp.dockTile.badgeLabel`).
- Launch at login: tauri-plugin-autostart, Settings toggle.
- Power assertion while any owned session is working: spawn
  `caffeinate -dims` and kill it when the fleet goes idle (subprocess beats
  IOKit ffi; debounce 60s so brief idles don't churn). This makes the
  "runs with the lid closed" chip honest.

## Definition of done

1. Main window closed entirely: a real owned session hits a pending
   approval → menu bar dot goes red with count, notification arrives;
   **Approve on the notification unblocks the real session** (transcript
   proof); the dot cycles red → yellow → green as it runs and finishes.
2. Reply… on a notification with "use the staging key instead" → request
   denied, guidance visible in the session transcript.
3. ⌥Space from another app (e.g. while a browser is focused): type `a` →
   pending approval resolves; type `3: summarize progress` → session 3 goes
   yellow.
4. Dropdown: approve from it, prompt from its input, click a session row →
   main window opens focused on that session.
5. `caffeinate` appears in `ps` while a session works, gone ~60s after idle.
6. compare.py OK · tsc · cargo test (incl. grammar tests) · tauri dev boots.
